────────────────────────────────────────────────────────────────────────────────
  ARCHITECTURE OVERVIEW
────────────────────────────────────────────────────────────────────────────────

  Single-file web application — the entire product lives in one index.html file
  with no build step, no bundler, and no compilation required. All dependencies
  are loaded from CDN at runtime. The app runs in the browser as a static file
  served by a lightweight local server.

  JavaScript is organised into isolated IIFE (Immediately Invoked Function
  Expression) modules that communicate through a shared window.* namespace.
  There is no module system (no import/export), no framework, and no virtual DOM.

────────────────────────────────────────────────────────────────────────────────
  FRONTEND — JAVASCRIPT MODULES (IIFEs)
────────────────────────────────────────────────────────────────────────────────

  The app is split into four IIFE script blocks, each responsible for one domain:

  1. Chat IIFE
     — Manages the right-panel AI chat UI
     — Sends messages to Anthropic API with streaming SSE
     — Maintains window.conversationHistory (in-memory [{role, content}] array)
     — Exposes window.addMessage() globally for cross-IIFE use
     — Disables input during streaming; shows "Thinking..." badge

  2. File Ingestion IIFE
     — Manages the left-panel Sources list
     — Handles drag-and-drop of individual files and folders
     — Recursive folder traversal via webkitGetAsEntry API
     — Folder picker via <input webkitdirectory>
     — Data model: sources[] containing {type:'file'} and {type:'folder'} objects
     — Exposes window.openFile() and window.closeFile() for cross-IIFE use

  3. Excel Editor IIFE
     — Manages the centre-panel spreadsheet
     — Parses .xlsx with SheetJS, renders with Jspreadsheet CE
     — Exposes window.openFile / window.closeFile / window._excelState
     — Handles .xlsx export/download via SheetJS write + Blob

  4. AI Consolidation IIFE (inline script block)
     — Reads checked source files, serialises to AoA via SheetJS
     — Calls Anthropic Messages API (non-streaming, structured JSON response)
     — Parses merged AoA + prose summary from response
     — Loads merged result back into Excel editor via window.openFile()
     — Posts summary to chat via window.addMessage()

  Shared window-scope state (declared between script blocks):
     window.ANTHROPIC_API_KEY       — API key shared by chat + consolidation
     window.conversationHistory     — Chat turn history [{role, content}]
     window.addMessage              — Chat bubble renderer
     window.openFile / closeFile    — Excel editor file lifecycle
     window._excelState             — {instance, workbook, fileName}

────────────────────────────────────────────────────────────────────────────────
  AI — ANTHROPIC API
────────────────────────────────────────────────────────────────────────────────

  Provider .............. Anthropic (api.anthropic.com)
  API version ........... 2023-06-01
  Model ................. claude-opus-4-5 (used for both chat and consolidation)
  Access method ......... Direct browser fetch() with header:
                          'anthropic-dangerous-direct-browser-access': 'true'
                          (No backend proxy — API key lives in window scope)
  API key storage ....... window.ANTHROPIC_API_KEY (hardcoded placeholder,
                          marked with // TODO: remove before sharing)

  Chat endpoint usage:
    — stream: true
    — SSE parsing via response.body.getReader() + TextDecoder
    — Tokens streamed token-by-token into a live chat bubble
    — Full conversation history sent on every request (multi-turn context)
    — Failed responses excluded from conversationHistory

  Consolidation endpoint usage:
    — stream: false (non-streaming — complete JSON must be parsed at once)
    — All checked .xlsx files read as ArrayBuffer, converted to AoA JSON
    — Prompt instructs Claude to merge into a single unified AoA
    — Response format: raw JSON array followed by "\nSUMMARY:" + prose paragraph
    — JSON parsed → SheetJS → synthetic .xlsx File → loaded into editor

────────────────────────────────────────────────────────────────────────────────
  KEY ARCHITECTURAL DECISIONS
────────────────────────────────────────────────────────────────────────────────

  Single HTML file ........ No build pipeline needed; easy to share and deploy;
                            entire product is one file anyone can open
  IIFE modules ............ Encapsulation without a module bundler; each domain
                            (chat, files, excel, AI) has its own scope
  window.* globals ........ Cross-IIFE communication without a message bus or
                            event system; explicit, readable surface area
  Direct browser API call . No backend required for v1; trades key security for
                            zero infrastructure; flagged for pre-production fix
  SheetJS + Jspreadsheet .. SheetJS handles all parse/write; Jspreadsheet handles
                            display/editing; clean separation of concerns
  No approval workflow .... Deferred — not needed for v1 per stakeholder decision
  .xlsx only .............. PDF/CSV deferred; .xlsx covers all subcontractor files
