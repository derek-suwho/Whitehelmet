## Naming Conventions

**Files:**
- `.mjs`: ES modules (serve.mjs, screenshot.mjs)
- `.html`: Web entry points (index.html)
- `.json`: Configuration and dependency manifests (package.json, package-lock.json)
- Screenshot outputs: `screenshot-N.png` (auto-incrementing integer)

**Directories:**
- camelCase: Logical grouping (node_modules, temporary screenshots)
- UPPERCASE NUMBERS: Categorical folders (Jul-2024, UC-2)
- Dot-prefix: Internal/hidden (`.git`, `.planning`, `.gitignore`)

**CSS Class Naming:**
- `.panel-{name}`: Main layout sections (panel-sources, panel-excel, panel-chat)
- `.{section}-{component}`: Sub-elements (chat-bubble, chat-input-box, panel-header)
- `.{component}-{state}`: Variant styling (chat-bubble-user, chat-bubble-ai)
- Modifier classes use descriptive names (empty-state, toolbar-btn, badge-amber)

**JavaScript Identifiers:**
- camelCase: Variable and function names (chatHistory, chatInput, sendMessage)
- CONSTANT_CASE: Constants (AI_MOCK_REPLY)
- ID-based selectors: document.getElementById('chatHistory'), etc.

## Where to Add New Code

**New Feature (e.g., file upload handler):**
- Primary code: Add script block in `<script>` section at end of `index.html` (before closing `</script>`)
- Integration point: Hook into existing event listeners or add new ones to drop zone
- Example: File parsing logic for .xlsx files should be in this same script tag

**New Panel/UI Component:**
- HTML markup: Add new `<aside>` or `<section>` with class `panel` in `.panels-container` grid
- Styles: Add CSS rules in `<style>` block with `.panel-{name}` class structure
- Behavior: Add event listeners in main script block
- Note: Adjust grid-template-columns on `.panels-container` if changing panel count

**Utilities or Helper Functions:**
- Small helpers: Inline in `<script>` block in index.html
- Reusable modules: Consider creating `.mjs` file and importing via `<script type="module">`
- Example: Excel parsing utility could become `excel-parser.mjs` and imported

**Development Scripts:**
- Location: Root of Whitehelmet directory
- Pattern: `.mjs` extension, use Node.js APIs
- Example: New automation script should follow screenshot.mjs pattern

**Styling Extensions:**
- Theme colors: Extend tailwind.config colors object (lines 11-35)
- Shadows: Add to boxShadow (lines 52-56)
- Other utilities: Add to theme.extend object (lines 10-57)
