<task type="auto">
  <name>Task 1: Wire upload handlers and file list rendering</name>
  <files>index.html</files>
  <action>
Add all file ingestion JS inside the existing IIFE (or a new IIFE below the chat one). Implementation:

1. **State**: Create an `uploadedFiles` array holding objects: `{ name, size, lastModified, file }`. The `file` property stores the raw File object for Phase 3 (Excel parsing). Use a Set or name-check to prevent duplicate uploads of same filename.

2. **Hidden file input**: Add a hidden `<input type="file" id="file-input" accept=".xlsx" multiple>` element just inside the body (before the script). Wire its `change` event to call `handleFiles(input.files)`.

3. **Drop zone wiring**: Get the existing `.drop-zone` element. Add `dragover` (preventDefault + set `dropEffect = 'copy'`), `dragleave`, and `drop` event listeners. On drop, call `handleFiles(e.dataTransfer.files)`. Also add the same listeners on the entire `.panel-left` so dropping anywhere on the panel works (not just the small drop zone area).

4. **Visual drag feedback**: On dragover, add a CSS class `.drag-active` to the drop zone that changes border to `rgba(226,154,53,0.6)` solid and background to `rgba(226,154,53,0.06)`. Remove on dragleave and drop.

5. **handleFiles(fileList)**: Iterate the FileList, filter to only `.xlsx` extension (case-insensitive check on `file.name.endsWith('.xlsx')`), skip duplicates by name, push to `uploadedFiles`, then call `renderFileList()`.

6. **renderFileList()**:
   - Clear panel-body innerHTML
   - If uploadedFiles is empty, restore the empty state HTML (clone from a template or re-create)
   - If files exist, render a list. Each file item is a div with class `source-file-item` containing:
     - A small Excel file icon (green-tinted SVG, 16x16)
     - File name (truncated with ellipsis if >25 chars, full name in title attribute)
     - Size in human-readable format (formatFileSize helper: bytes/KB/MB)
     - Last modified date formatted as "MMM D, YYYY" (use file.lastModified, format with toLocaleDateString)
   - Update the `.panel-header-badge` text to show "{N} file{s}"
   - Style each file item: padding 10px 14px, border-bottom 1px solid surface-border color, hover background surface-muted. Use the existing color tokens from the Tailwind config (surface.panel, surface.border, content.secondary for metadata text).

7. **Click to browse**: Wire the existing `.drop-zone` click to trigger `fileInput.click()`. Also wire the "or click to browse" text.

8. **Add CSS** for `.source-file-item` (hover state, layout), `.drag-active` class, and file metadata styling. Use construction palette colors consistent with Phase 1. File names in content.primary color, metadata (size/date) in content.secondary, smaller font.
