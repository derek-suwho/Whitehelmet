<task type="auto">
  <name>Task 1: Add folder drop support via webkitGetAsEntry</name>
  <files>index.html</files>
  <action>
Modify the existing drop event handler in index.html to detect directory entries and recursively extract .xlsx files. Implementation:

1. **Update the drop handler**: Before calling handleFiles, check if `e.dataTransfer.items` is available. If so, iterate items and call `item.webkitGetAsEntry()` on each. If any entry is a directory (`entry.isDirectory`), route to the async traversal path instead of the simple handleFiles path.

2. **traverseDirectory(directoryEntry)**: Create an async function that:
   - Creates a DirectoryReader via `directoryEntry.createReader()`
   - Reads entries in batches (readEntries returns max ~100 at a time, must loop until empty batch)
   - For each entry: if `isFile`, call `entry.file(cb)` to get the File object, collect it. If `isDirectory`, recurse.
   - Returns a Promise resolving to an array of all File objects found recursively.

3. **handleDrop integration**: When folder entries detected, collect all files from all dropped items (mix of files and folders), then pass the collected File array to `handleFiles()`. Make sure handleFiles can accept a plain Array (not just FileList) — if 02-01 used `Array.from(fileList)` internally this already works, otherwise add that conversion.

4. **Edge case**: If the browser doesn't support webkitGetAsEntry (rare but possible), fall back to the existing `e.dataTransfer.files` behavior which works for individual files.

5. **No UI changes needed** — the drop zone visual, drag-active class, and file list rendering from Plan 01 all work as-is since folder contents are just more File objects fed through handleFiles.

Note: webkitGetAsEntry is supported in Chrome, Edge, Firefox, and Safari. The "webkit" prefix is standard — it's the accepted cross-browser API per the File and Directory Entries API spec.
  </action>
