Create the project infrastructure and layout:

1. **serve.mjs** — Simple Node.js static file server on port 3000 serving the project root. Use built-in `http` and `fs` modules (no dependencies). Handle common MIME types (html, css, js, png, jpg, svg). Default to index.html for root requests.

2. **screenshot.mjs** — Puppeteer screenshot utility. Takes URL as first arg, optional label as second arg. Saves to `./temporary screenshots/screenshot-N.png` (auto-incremented). Uses Chrome cache at `/Users/dereksu/.cache/puppeteer/`. Viewport 1280x800.

3. **index.html** — Single-file app with all styles inline via Tailwind CDN:
   - Three-column layout using CSS Grid: `grid-cols-[280px_1fr_320px]`
   - Full viewport height (`h-screen`), no scroll on the outer container
   - **Left panel (Sources):** Header "Sources", empty state placeholder text "Drop .xlsx files here", subtle left border styling
   - **Center panel (Excel):** Header "Excel Editor", empty state placeholder "Select a source file to view", this is the widest panel
   - **Right panel (Chat):** Header "AI Chat", empty state placeholder "Chat will appear here", subtle right border styling
   - Each panel: distinct background shade, panel header with label, internal scroll if content overflows
   - Apply CLAUDE.md anti-generic guardrails: custom brand colors (NOT default Tailwind blue/indigo — use a construction-themed palette, e.g., slate/amber/warm tones), paired typography (display font for headings, clean sans for body), layered shadows with color tint, intentional spacing tokens, surface depth system (base bg, elevated panels, floating elements)
   - No animations or interactive states yet beyond panel structure
  </action>
