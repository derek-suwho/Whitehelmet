Modify the right panel ("AI Chat") in index.html to contain a fully functional chat interface:

1. **Layout structure** inside the chat panel (flex column, full height):
   - Top: Panel header "AI Chat" (already exists from Plan 01)
   - Middle: Scrollable message history area (`flex-1 overflow-y-auto`). This area grows to fill available space.
   - Bottom: Fixed input bar with text input + send button, pinned to bottom of panel

2. **Message history area:**
   - Container for chat messages, scrolls vertically when content overflows
   - Empty state: subtle text "Start a conversation..." centered in the area
   - When messages exist, render them as chat bubbles:
     - **User messages:** Right-aligned, branded accent color background, white text
     - **AI messages:** Left-aligned, lighter/muted background, dark text
   - Each message shows the text content. No timestamps needed yet.
   - Auto-scroll to bottom when new message added

3. **Input bar:**
   - Text input field: full width minus send button, placeholder "Type a message...", border with focus ring
   - Send button: icon or text "Send", branded accent color, hover/focus/active states per CLAUDE.md guardrails
   - Send triggers on: button click OR Enter key press (Shift+Enter for newline NOT needed yet)
   - On send: read input value, if non-empty create a user message bubble in history, clear input, refocus input
   - After user message, add a mock AI response after 500ms delay: "I'm an AI assistant for Whitehelmet. AI responses will be connected in a future update." (This proves the message rendering pipeline works for both sides.)

4. **Styling:**
   - Match the color palette from Plan 01's layout
   - Chat bubbles: rounded corners, appropriate padding, max-width ~80% of chat area
   - Input bar: subtle top border separating it from message area
   - Send button: interactive states (hover, focus-visible, active) — animate only `transform` and `opacity`, never `transition-all`
   - Smooth scroll behavior on message area

5. **JavaScript** (inline `<script>` at end of body):
   - `sendMessage()` function: reads input, validates non-empty, creates user bubble DOM element, appends to history, scrolls to bottom, clears input, schedules mock AI reply
   - `addMessage(text, sender)` helper: creates a message element with appropriate alignment/color based on sender ("user" or "ai"), appends to message history container
   - Event listeners: click on send button, keydown Enter on input
